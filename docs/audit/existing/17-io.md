# io/ — atomicfile, progress

## Landed

- ✅ **atomicfile preserves file mode** — `Save` now Stats the destination before write, Chmods the temp to match, so replacing a 0600 secret no longer leaves it world-readable (commit `c502dd2`).
- ✅ **atomicfile EXDEV fallback** — build-tagged `exdev_unix.go` / `exdev_windows.go` expose the platform's cross-device error; `Save` falls back to copy+rename when temp and target are on different filesystems (common in containers with tmpfs `TMPDIR`) (commit `c502dd2`).

## Open

_Closed — see Recently Landed below._

## Recently Landed (Phase 3)

- ✅ **progressReader / throttledReader concurrency doc** — both types now carry an explicit "NOT safe for concurrent Read" doc comment, matching the conventional `io.Reader` contract.
- ✅ **progressReader callback coalescing** — `WithThrottle(d)` and `WithMinDelta(bytes)` options coalesce callbacks; the final EOF callback is always delivered so consumers see the completed total.
